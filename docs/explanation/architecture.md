# How The Pieces Fit Together

Longform Kit uses each tool for the part it already solves well. It does not
replace Quarto's publishing model with another one.

## Zettlr Is The Writing Environment

Zettlr opens `document/` as the project, lists the ordered Markdown sources,
and helps insert Zotero citations. Authors point Zettlr's citation preferences
at the same Better CSL JSON export used by Longform Kit. The generated
`document/.ztr-directory` is UI state derived from Quarto, not a second build
manifest.

Author-maintained files under `document/` are ordinary `.md`, alongside
`document/metadata.yml` (title, author, date, language) and
`document/chapters.yml` (the ordered chapter list). The root `index.md` exists
only because a Quarto book requires a home page; it includes
`document/front-matter.md` and is regenerated by `bin/longform setup`.

## Quarto Is The Project And Build Layer

Root `_quarto.yml` is the small project-discovery loader Quarto requires. Its
`metadata-files` list merges `quarto/project.yml`, `document/metadata.yml`, and
`document/chapters.yml`. The substantive `quarto/project.yml` provides:

- The standard `book` project type.
- Stable bibliography and CSL link paths, the output basename, and
  `metadata-files` links to the author-owned `document/metadata.yml` (title,
  author, date, language) and `document/chapters.yml` (ordered chapter
  manifest).
- Native PDF, DOCX, and LaTeX formats.
- Native citeproc, TOCs, page breaks, conditional content, and profiles.
- A machine-readable resolved configuration through `quarto inspect`.
- A pinned bundled Pandoc.

Binding is still selected as Quarto's `binding` profile, so profile-conditional
content behaves normally. Because the override is organized at
`quarto/binding.yml` rather than root `_quarto-binding.yml`, `bin/longform`
also passes it as an explicit metadata file. Bare
`quarto render --profile binding` does not load the relocated override. DOCX
styling is an ordinary `reference-doc`. There is no Longform-specific Quarto
project type or format.

## Pandoc Is The Document Engine

Pandoc parses Markdown, processes citations, and writes the selected format.
Longform Kit leaves citeproc under Quarto's control. Epigraphs use Andrew
Heiss's vendored Fancy Epigraphs v0.0.1 shortcode; it writes LaTeX epigraphs for
PDF/LaTeX and portable blockquotes for formats such as DOCX and GFM.

## Zotero Is The Bibliographic Authority

Zotero owns editable item metadata and installed citation styles. Better
BibTeX keeps an external Better CSL JSON export current. `bin/longform setup`
links that file and the selected installed style to stable ignored paths under
`references/`; it also links `references/zotero-styles` to Zotero's installed
style directory. Quarto and Pandoc therefore see Zotero and Better BibTeX
updates without copying citation files into Git.

Each machine and CI checkout must run setup. Zettlr must also be pointed
manually to the same Better CSL JSON export.

## The CLI Is The Stable Contract

`bin/longform` gives authors, CI, Zettlr, and provider-agnostic AI agents the
same commands. It links local Zotero inputs, validates paths and citations,
synchronizes Zettlr, checks fonts, invokes native formats, and normalizes output
names.

```text
External Better CSL JSON -------> Zettlr citation preferences
           |
           +---------------------+
                                 |
Zotero installed styles --------+----> ignored references/ links
                                               |
Zettlr edits document/**/*.md -----------------+
                                               |
                                               v
_quarto.yml loads quarto/project.yml + document/*.yml
                       |
                       v
                 bin/longform
                       |
                       v
            Quarto + Pandoc + native citeproc
                 /             |             \
                v              v              v
       PDF / DOCX / LaTeX  binding PDF  standalone GFM
                                  ^
                                  |
                       quarto/binding.yml
```

GFM is the only assembly exception because Quarto does not offer GFM as a
combined book format. The CLI concatenates the resolved author sources into a
temporary standalone Quarto document outside the book project and mirrors the
project's resource paths there. Quarto still expands includes, Fancy Epigraphs,
native page breaks, format conditionals, and citations. Pandoc extracts
embedded images to `<output-name>_files/`, and both the Markdown and images are
copied to `build/`.
