# How The Pieces Fit Together

Longform Kit uses each tool for the part it already solves well. It does not
replace Quarto's publishing model with another one.

## Zettlr Is The Writing Environment

Zettlr opens the repository as a project, lists the ordered Markdown sources,
and helps insert Zotero citations. The generated root `.ztr-directory` is UI
state derived from Quarto, not a second build manifest.

Every author-maintained file under `document/` is ordinary `.md`. The root
`index.md` exists only because a Quarto book requires a home page; it includes
`document/front-matter.md` and is regenerated by `bin/longform setup`.

## Quarto Is The Project And Build Layer

Root `_quarto.yml` provides:

- The standard `book` project type and ordered chapter manifest.
- Shared metadata, bibliography, CSL, and output basename.
- Native PDF, DOCX, and LaTeX formats.
- Native citeproc, TOCs, page breaks, conditional content, and profiles.
- A machine-readable resolved configuration through `quarto inspect`.
- A pinned bundled Pandoc.

The binding PDF is an ordinary Quarto profile. DOCX styling is an ordinary
`reference-doc`. There is no Longform-specific Quarto project type or format.

## Pandoc Is The Document Engine

Pandoc parses Markdown, processes citations, and writes the selected format.
Longform Kit leaves citeproc under Quarto's control. Epigraphs use Andrew
Heiss's vendored Fancy Epigraphs v0.0.1 shortcode; it writes LaTeX epigraphs for
PDF/LaTeX and portable blockquotes for formats such as DOCX and GFM.

## Zotero Is The Bibliographic Authority

Zotero owns editable item metadata. Better BibTeX exports a project collection
to `references/library.json`, and `references/style.csl` pins citation
rendering. Both files are local, so routine builds do not need Zotero or the
network.

## The CLI Is The Stable Contract

`bin/longform` gives authors, CI, Zettlr, and provider-agnostic AI agents the
same commands. It validates paths and citations, synchronizes Zettlr, checks
fonts, invokes native formats, and normalizes output names.

```text
Zettlr edits document/**/*.md
              |
              v
_quarto.yml orders the standard book
              |
              v
Quarto + Pandoc + native citeproc
              |
              +----> PDF / binding PDF / DOCX / LaTeX
              |
              +----> temporary standalone GFM render
```

GFM is the only assembly exception because Quarto does not offer GFM as a
combined book format. The CLI concatenates the resolved author sources into a
temporary standalone Quarto document outside the book project and mirrors the
project's resource paths there. Quarto still expands includes, Fancy Epigraphs,
native page breaks, format conditionals, and citations. Pandoc extracts
embedded images to `<output-name>_files/`, and both the Markdown and images are
copied to `build/`.
